广度优先搜索

二月 10, 2020 发布在 算法学习笔记, 算法

功能

广度优先搜索算法（BFS）主要应用于图的遍历，也可以找到当前节点到目标节点的最短路径（每条边等价的前提下）。

时间复杂度

对于一个有n个节点，e条边的图，时间复杂度为O(n + e)。

原理

对于一个图，首先确定一个起点，先遍历起点的每一个邻节点，然后再遍历每个邻节点的邻节点，以此类推，直到每一个节点都被检查过。

具体的实现可以通过队列（利用其先入先出的性质），用队列来存放待检查的节点。首先把起点的所有邻节点放入队列，之后检查到每一个节点时，都把该节点的所有邻节点加入队列。每次检查只检查位于队首的节点，检查后将该节点从队首移除。由于邻居一定比邻居的邻居先被加入队列，所以也会被优先处理。直到队列为空。

但还有个问题，当图中含有环的时候，可能导致死循环。因此还需要用一个列表存放所有已经被遍历过的节点。

Python实现

from collections import deque

graph = {}
graph["you"] = ["alice", "bob", "claire"]
graph["bob"] = ["anuj", "peggy"]
graph["alice"] = ["peggy"]
graph["claire"] = ["thom", "jonny"]
graph["anuj"] = []
graph["peggy"] = []
graph["thom"] = []
graph["jonny"] = ["you"]

def bfs(graph, person_to_find):
    search_queue = deque()
    checked_queue = []
    search_queue += graph["you"]

    while search_queue:
        person = search_queue.popleft()
        if person == person_to_find:
            return True
        elif person not in checked_queue:
            search_queue += graph[person]
            checked_queue.append(graph[person])
    return False

print(bfs(graph, "jonny"))
print(bfs(graph, "adam"))

以上代码只实现了搜索，但没有求出具体的最短路径。如果要求出具体的路径，可以新建一个字典，存放每一个节点的前一个节点是哪个，然后在广搜完成后回溯得到具体的路径。
