动态规划

二月 17, 2020 发布在 算法学习笔记, 算法

功能

动态规划是一种求解决策过程最优化的算法。什么意思呢？先举个例子。

假如你是一个小偷，你的背包可以装下4磅的东西。现在可以偷的东西有三样：音响（3000元，4磅）、笔记本电脑（2000元，3磅）还有吉他（1500元，1磅）。

怎么偷最赚？（当个小偷也不容易）

这就是著名的01背包问题。这个问题就属于决策过程最优化问题，可以用动态规划解决。实际上，决定一个问题能否用动态规划解决，关键是看这个问题能否被拆分成小问题，且每个小问题是否能被重复利用。

原理

还是用刚才小偷的例子，如何把这个问题拆分成小问题呢？一个大背包可以分成几个小背包，因此你可以先解决子背包的问题。动态规划算法都从一个网格开始，背包问题的网格如下：

1 2 3 4
吉他
音响
笔记本电脑

1、2、3、4表示的是子背包的容量，各行则为不同的商品。第一行表示你只能偷吉他，第二行表示你只能偷吉他和音响，第三行表示你都可以偷。要填的就是每个格子对应的最大价值。将这个表格填完：

1 2 3 4
吉他 1500 1500 1500 1500
音响 1500 1500 1500 3000
笔记本电脑 1500 1500 2000

右下角这个空怎么填？此时背包容量为4磅，而且三样物品都可以偷。首先，假如不偷笔记本电脑，从第二行第四列我们可知，最优解是3000。假如偷笔记本电脑，那么剩余的空间是4-3=1磅。从第二行第一列可知，对于1磅的空间且只能拿吉他或音响的情况下，最优解是1500。所以偷笔记本电脑能得到的价值是2000 + 1500 = 3500 > 3000，因此最优解为3500。

其实每个格子的值都可以利用公式：max(cell[i - 1][j], 当前商品价值 + cell[i - 1][j - 当前商品的重量])，简单来说就是当前商品拿或者不拿，怎样更赚。有了公式，实现起来就很容易了。上面的这个网格被称作动态规划矩阵（dp matrix）。

当然，01背包问题只是动态规划的其中一个应用。动态规划更多提供的是一种解决问题的思路。实现动态规划的关键是写出dp矩阵。

Python实现

def dp(weight, value, capacity):
    row = len(weight)
    column = capacity
    dp_matrix = [[0 for i in range(column+1)] for j in range(row+1)]

    for i in range(1,row+1):
        for j in range(1,column+1):
            if weight[i-1]<=j:
                dp_matrix[i][j] = max(dp_matrix[i-1][j],dp_matrix[i-1][j-weight[i-1]]+value[i-1])        #compare which option is better: take this item or not

            else:
                dp_matrix[i][j] = dp_matrix[i-1][j]       #if the capacity for this column is not big enough, you can't take the item.

    return dp_matrix

def find_items(weight, capacity, dp_matrix):        #this function is used to find what items have been taken

    row = len(weight)
    flag = [0 for i in range(row)]
    j = capacity
    for i in range(row,0,-1):
        if dp[i][j] != dp[i-1][j]:
            flag[i-1] = 1
            j -= weight[i-1]
    return flag

weight = [1, 4, 3]
value = [1500, 3000, 2000]
dp = dp(weight, value, 4)
print("The highest value is " + str(dp[3][4]))
print(find_items(weight, 4, dp))

局限性

动态规划虽好，但也有它的局限性。刚才的01背包问题，这个01表示什么呢？其实意思就是说，这些物品只能拿或者不拿，没有"拿一部分"的选项。对于可以"拿一部分"的问题，不能采用动态规划。

其次，将问题拆解为小问题后，每个小问题之间必须是相互独立的。也就是说，之前发生的事不能够影响后面的事。举个例子，假如说我偷了A物品，再去偷B物品，那么B物品的价值就会提升，那这种情况也不能用动态规划，因为是否偷取A，会对B的价值造成影响。
